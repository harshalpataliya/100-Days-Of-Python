# Day 17 - The Quiz Game (Object-Oriented Programming)

## Overview

Today I built a Quiz Game using Object-Oriented Programming (OOP). I learned how to create my own classes, initialize objects using constructors, and organize program logic using methods. Instead of writing everything in one file, I separated the project into multiple files to make the code cleaner and easier to manage.

## What I Learned

- Creating custom classes
- Constructors (`__init__()`)
- Initializing object attributes using `self`
- Creating objects from classes
- Understanding the difference between classes and objects
- Creating and using methods inside a class
- Working with object attributes
- Storing and managing objects inside a list
- Passing objects between different classes
- Organizing code into multiple Python files
- Building a quiz application using OOP principles


- Loads quiz questions from a separate data file
- Creates `Question` objects from the question data
- Stores all questions in a question bank
- Displays one question at a time
- Accepts user input (True/False)
- Checks whether the answer is correct
- Keeps track of the player's score
- Displays the final score after all questions are completed

## Files

- `main.py` – Starts the quiz and controls the program flow
- `data.py` – Stores all quiz questions
- `question_model.py` – Defines the `Question` class
- `quiz_brain.py` – Handles quiz logic, scoring, and question progression

## OOP Concepts Practiced

- Classes
- Objects
- Constructors (`__init__`)
- Attributes
- Methods
- Object Interaction
- Encapsulation
